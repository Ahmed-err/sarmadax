## How to Give Feedback

Good feedback makes the project faster and better. Here's how to do it:

### Do ✓
- Be specific: *"The button on the pricing card should be teal, not blue"* not *"change some colors"*
- Reference the element: use screenshots with annotations if helpful
- Consolidate feedback: send it all in one message per review round, not drip-by-drip
- Prioritize: mark what's a blocker vs. what's a nice-to-have

### Don't ✗
- Don't approve something and then change your mind later (revision rounds are finite)
- Don't add new features mid-review — note them and we'll discuss as potential scope additions
- Don't go silent — if you need more time, just let me know

### Feedback format (suggested)
```
Page / Screen: [e.g., Homepage hero]
Element: [e.g., Headline text]
Current: [what it says/looks like now]
Requested change: [what you want instead]
Priority: [must have / nice to have]
```
